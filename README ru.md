# DDNet Chat Translator

Модификация клиента [DDNet](https://ddnet.org/) — встроенный переводчик чата на базе DeepL API. Переводит входящие сообщения прямо в игре и позволяет отвечать на выбранном языке, без переключения на сторонние переводчики.

> Работает **только на стороне клиента** — это не серверный мод. Функция активна только у того, кто собрал/скачал этот клиент, и работает на любом сервере DDNet, независимо от того, установлена ли эта модификация у остальных игроков.

## Возможности

- Автоматический перевод входящих **личных сообщений (whisper)**.
- Автоматический перевод сообщений, где **упоминается ваш ник**.
- Режим **"переводить весь чат"** — по желанию, одной галочкой.
- Отметка **конкретного игрока**, чьи сообщения переводятся всегда, независимо от остальных настроек.
- Перевод и отправка **собственного ответа** — лично игроку или в общий чат, на любом выбранном языке.
- Умное подавление дублей: если сообщение и так на языке, который вы читаете, перевод не показывается повторно.
- Список языков подгружается напрямую из DeepL, с полем поиска — не нужно помнить коды языков.

## Требования

Для работы перевода нужен **личный API-ключ DeepL**. Без ключа переводчик просто ничего не делает — остальной клиент работает как обычный DDNet.

### Как получить ключ

1. Зарегистрируйтесь на [deepl.com/pro-api](https://www.deepl.com/pro-api) — есть бесплатный тариф (Free), ключ для него оканчивается на `:fx`.
2. Скопируйте выданный ключ из личного кабинета DeepL.
3. Вставьте его в клиенте: **Settings → Translator → DeepL API key**.

Актуальные лимиты бесплатного тарифа (объём символов в месяц и условия) могут меняться — уточняйте их в своём личном кабинете на сайте DeepL.

**Свой ключ никому не показывайте и не публикуйте** — он привязан к вашему аккаунту DeepL и вашей квоте.

## Установка / сборка

```bash
sudo apt install build-essential cargo cmake git glslang-tools google-mock \
  libavcodec-extra libavdevice-dev libavfilter-dev libavformat-dev libavutil-dev \
  libcurl4-openssl-dev libfreetype6-dev libglew-dev libnotify-dev libogg-dev \
  libopus-dev libopusfile-dev libpng-dev libsdl2-dev libsqlite3-dev libssl-dev \
  libvulkan-dev libwavpack-dev libx264-dev ninja-build python3 rustc spirv-tools

git clone --recursive https://github.com/<ваш-профиль>/ddnet
cd ddnet
mkdir build && cd build
cmake ..
make -j$(nproc)
```

Готовый бинарник: `build/DDNet`.

> Если сборка падает с внутренней ошибкой компилятора (`internal compiler error`) — известный баг новых версий GCC при компиляции `<format>`. Собирайте через Clang:
> ```bash
> cmake -DCMAKE_C_COMPILER=clang -DCMAKE_CXX_COMPILER=clang++ ..
> make -j$(nproc)
> ```

## Настройка после установки

Откройте **Settings → Translator** в меню клиента:

<!-- Сюда же можно продублировать скриншот вкладки настроек с подписанными полями -->
![Настройка вкладки Translator](screenshots/settings-tab-annotated.png)

1. **Enable chat translation** — включает перевод.
2. **DeepL API key** — вставьте свой ключ (см. раздел «Требования» выше).
3. **Reading language** — язык, на который переводятся входящие сообщения. Список подгружается из DeepL, есть поиск.
4. **Reply language** — язык, на который переводится ваш собственный текст при использовании команд `/tw` и `/twa`.
5. **Translate all chat messages** — включите, если хотите переводить вообще весь чат, а не только личные сообщения и упоминания вашего ника.

## Команды

Работают **и в игровом чате, и в консоли** (`~`) — можно вводить как обычную команду в консоли, так и прямо в поле чата с ведущим `/`.

| Команда | Где работает | Что делает |
|---|---|---|
| `/tr текст` | чат и консоль | Переводит `текст` на **Reading language** и показывает результат только вам — никому не отправляется. Удобно, чтобы быстро прочитать иностранную фразу или проверить, как она звучит на другом языке. |
| `/tw Игрок текст` | чат и консоль | Переводит `текст` на **Reply language** и отправляет его указанному игроку личным сообщением (аналог `/w`). |
| `/twa текст` | чат и консоль | Переводит `текст` на **Reply language** и отправляет его в общий чат, всем. |

Все три команды сохраняются в истории чата — можно вызвать их повторно стрелкой **вверх**, как и обычные сообщения.

**Пример:**
```
/tw Player123 Привет, как дела?
```
Игрок `Player123` получит личное сообщение с переводом вашей фразы на язык, выбранный в **Reply language**.

## Отметка отдельных игроков

На вкладке **Players** (список игроков сервера — там же, где кнопки мьюта и добавления в друзья) есть колонка **Translate**.

<!-- Скриншот крупным планом: строка игрока с включённым переключателем Translate -->
![Переключатель Translate у игрока](screenshots/player-translate-toggle.png)

Включив переключатель напротив конкретного игрока, вы будете получать перевод **всех** его сообщений в обычном чате — даже если общий режим "переводить весь чат" выключен, и даже если он не упоминает ваш ник напрямую.

> Отметка действует только в рамках текущего захода на сервер и сбрасывается при переподключении — в отличие от списка друзей, она не сохраняется постоянно.

## Как это работает "под капотом" (коротко)

1. Сервер присылает клиенту сообщение чата (личное или обычное).
2. Клиент проверяет условия перевода: whisper? упоминание ника? включён режим "весь чат"? отмечен ли отправитель галочкой Translate?
3. Если условие выполнено — текст уходит асинхронным HTTP-запросом в DeepL API, не блокируя игру.
4. Когда приходит ответ, определённый DeepL исходный язык сравнивается с вашим **Reading language** — если они совпадают, перевод не показывается (чтобы не дублировать одинаковый текст).
5. Иначе перевод печатается отдельной строкой в чате, отдельно от оригинала.
6. Команды `/tr`, `/tw`, `/twa` работают по той же схеме, но с текстом, который вводите вы сами, и с **Reply language** вместо Reading language.

## Известные ограничения

- Нужен собственный API-ключ DeepL — без него функция не работает вовсе.
- Лимиты DeepL Free (объём переводимых символов) действуют по вашему аккаунту — при интенсивном использовании (например, перевод всего чата на многолюдном сервере) квота расходуется быстрее.
- Перевод собственных сообщений работает только через явные команды `/tw` / `/twa`, а не автоматически при обычной отправке через Enter.
- Отметка "переводить этого игрока всегда" не сохраняется между заходами на сервер.
